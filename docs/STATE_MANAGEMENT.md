
# Управление состоянием (Zustand + Persistence)

## Архитектура хранилища
Приложение использует **Zustand** для реактивного управления состоянием. Поток данных организован таким образом, чтобы гарантировать синхронизацию между игровым полем, таймерами и модальными окнами.

### Разделение логики сброса:
В версии 2.6 реализовано четкое разделение между двумя типами сброса:
- **`resetGame`**: Возвращает приложение в "девственное" состояние настроек. Очищает текущую игру, но сохраняет библиотеку и конфигурацию ИИ.
- **`finishGame`**: Переводит игру в состояние `isGameOver: true`. При этом принудительно сбрасываются все флаги инструментов (`showEditor`, `showLibrary` и т.д.), чтобы гарантировать отображение `VictoryScreen` поверх игрового поля.

### Система временных данных (`temp`):
Объект `temp` в хранилище отвечает за транзитные состояния:
- `isProcessing`: Блокировка кликов во время анимации карт.
- `showLevelSelect` / `showQuestionModal`: Управление видимостью игровых модалок.
- `currentQuestion`: Активный вопрос, выбранный из пула набора.

## Механизм TTL (Time To Live)
Для предотвращения переполнения LocalStorage:
- Каждому пользовательскому набору присваивается `expiresAt` (Timestamp создания + 72 часа).
- Метод `cleanupLibrary` вызывается при инициализации приложения, удаляя просроченные наборы.
